Objective

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

EndPoints

Create a new user

Endpoint: POST /users

Explanation: This endpoint allows creating a new user with a unique id and name. The request should contain the user's name in the request body. If the request is successful, the response will contain the newly created user object with an autogenerated id. Example: POST /users with request body {"name": "Alice"}

Update a user

Endpoint: PUT /users/:id

Explanation: This endpoint allows updating an existing user's name. The request should contain the user's id in the request URL and the new name in the request body. If the request is successful, the response will contain the updated user object. Example: PUT /users/1 with request body {"name": "Bob"}

Add a favorite movie to a user's list

Endpoint: POST /users/:id/:movieTitle

Explanation: This endpoint allows adding a favorite movie to an existing user's list. The request should contain the user's id and the movie title in the request URL. If the request is successful, the response will contain a success message. Example: POST /users/1/Inception

List all movies

Endpoint: GET /movies

Explanation: This endpoint allows listing all movies in the system. If the request is successful, the response will contain an array of movie objects. Example: GET /movies

Search movies by title

Endpoint: GET /movies/:title

Explanation: This endpoint allows searching for a movie by its title. The request should contain the movie title in the request URL. If the movie is found, the response will contain the movie object. Example: GET /movies/Inception

List all movies in a specific genre

Endpoint: GET /movies/genre/:genreName

Explanation: This endpoint allows listing all movies in a specific genre. The request should contain the genre name in the request URL. If the genre is found, the response will contain the genre object with a list of movies. Example: GET /movies/genre/Action

Get all movies by a specific director

Endpoint: GET /movies/director/:directorName

Explanation: This endpoint allows listing all movies by a specific director. The request should contain the director name in the request URL. If the director is found, the response will contain the director object with a list of movies. Example: GET /movies/director/Christopher Nolan

Delete a favourite movie from a user's list

Endpoint: DELETE /users/:id/:movieTitle

Explanation: This endpoint allows deleting a favorite movie from an existing user's list. The request should contain the user's id and the movie title in the request URL. If the movie is found and deleted, the response will contain a success message. Example: DELETE /users/1/Inception

Delete user

Endpoint: DELETE /users/:id

Explanation: This endpoint allows deleting an existing user. The request should contain the user's id in the request URL. If the user is found and deleted, the response will contain a success message. Example: DELETE /users/1
